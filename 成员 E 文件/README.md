## 任务二：性能评测

### 2.1 评测目标

量化系统性能指标，评估不同 ANN 算法的效率和精度。

### 2.2 评测指标

| 指标 | 定义 | 计算方式 | 说明 |
|------|------|---------|------|
| **查询响应时间** | 单次查询耗时 | 平均值 (ms) | 越低越好 |
| **吞吐量** | 每秒处理查询数 | QPS = 1000 / 平均耗时 | 越高越好 |
| **Recall@K** | 召回率 | ANN 结果 ∩ 暴力搜索结果 / K | 越高越好 (100% 最优) |
| **索引构建时间** | 索引构建耗时 | 秒 (s) | 越低越好 |
| **索引内存占用** | 索引文件大小 | MB | 越低越好 |

### 2.3 运行性能评测

```bash
# 进入项目根目录
cd Software-Engineering-Major-Project-main/Software-Engineering-Major-Project-main

# 运行性能评测脚本
python "成员 E 文件/performance_benchmark.py"
```

## 常见问题

### Q1：FAISS 安装失败

```bash
# 尝试指定版本
pip install faiss-cpu==1.8.0.post1

# 或使用 conda 安装
conda install -c conda-forge faiss-cpu
```

### Q2：Flask 端口被占用

```bash
# Windows 查看端口占用
netstat -ano | findstr :5000

# 杀死占用进程
taskkill /F /PID <PID>
```

### Q3：测试数据不存在

确保 `成员B文件/data/cell_vectors.npy` 文件存在。如不存在，请先运行：

```bash
cd 成员B文件/code
python preprocessing.py
```

### Q4：前端显示 "Backend Offline"

1. 确保后端服务已启动：`cd 成员C文件 && python app.py`
2. 检查端口 5000 是否被占用
3. 确保通过 `http://localhost:8000` 访问，不要直接打开本地文件

### Q5：性能测试结果差异大

性能测试结果受以下因素影响：
- CPU 性能
- 内存大小
- 系统负载
- 数据规模

建议在相同环境下多次测试取平均值。
