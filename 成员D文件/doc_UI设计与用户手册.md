## 1. UI 设计与架构说明

### 1.1 前端技术选型与对齐情况
本项目前端设计采用了轻量化、高交互性的 B/S 架构，并与后端的 API 接口设计（成员 C）以及数据产出（成员 B）进行了严格对齐：
- **核心框架**：`Vue 3`（响应式数据绑定，实时更新检索参数与状态）。
- **组件库**：`Element Plus`（提供规范化、美观的表单、数据表格与全局排布）。
- **数据可视化**：`ECharts`（底层 Canvas 渲染，启用 `large: true` 模式应对 6.9w+ 数据点的散点着色绘制）。
- **数据对齐**：使用 `PapaParse` 在前端直接解析成员 B 产出的 `cell_2d_coords.csv`，将 `umap_1` 与 `umap_2` 映射为坐标轴，`cell_type` 映射为分类图例颜色。

### 1.2 界面布局布局设计
界面采用了经典的左右分栏布局（左侧操作面板，右侧大屏可视化）：
1. **顶部导航栏** (Header)
   - 系统标题与版本标记。
   - **后端状态监控标签**：实时感知成员 C 开发的后端状态。当监测到 `/api/dataset/info` 通信畅通时，亮起绿灯；否则提示离线。
2. **左侧交互控制面板** (Control Panel)
   - **参数输入区**：包括目标细胞索引选取（0~69031）、Top-K 返回数滑动条、距离计算度量（L2 欧氏距离 / Cosine 余弦相似度）。
   - **条件过滤区**：级联下拉框。先选择过滤字段（如 `cell_type`），前端会调用后端接口 `/api/dataset/column/{field}/values` 动态拉取有效取值，防止用户输入非法条件。
3. **左下角检索结果展示区** (Result Table)
   - 通过 `el-table` 可视化呈现后端返回的 ANN 检索结果，包含相似度排名（Rank）、细胞编号（Index）、距离得分（Distance）与核心属性。
4. **右侧全屏可视化视图** (UMAP Chart)
   - 展示六万余个细胞在二维空间的聚集聚类表征。

## 2. 用户操作手册

### 2.1 系统启动与访问
1. **启动后端服务**：确保已在含模型数据的机器上运行了 Flask 服务（默认运行于 `127.0.0.1:5000`）。
2. **启动页面服务**：在项目根目录运行 `python -m http.server 8000`。
3. **访问系统**：在现代浏览器中打开 `http://localhost:8000/成员D文件/index.html`。页面加载后会自动获取全部数据点，渲染 UMAP 散点群图约需要 2-3 秒，请耐心等待。

### 2.2 执行一次标准的 Top-K 检索
1. 在左侧面板的 **“目标细胞 Index”** 输入框中填写你感兴趣的细胞编号（例如 `1024`）。
2. 拖动 **“Top-K 返回数量”** 滑块，决定需要找出的最相似细胞的数量（例如 `20`）。
3. 点击 **“立即检索”** 按钮。
   > **视觉反馈**：系统会自动在左下方的表格中列出最相似的 20 个细胞信息。同时，右侧的 UMAP 图表中会发生标注变化：**🔴 红色大圆点**代表你的目标查询细胞，**🟡 带有黑色边框的黄色圆标**代表检索出的相似细胞。

### 2.3 执行条件过滤检索
如果你只希望在特定的细胞类型中寻找相似细胞：
1. 在 **“属性过滤”** 的第一个下拉菜单选择你想约束的字段，例如 `cell_type`。
2. 系统加载好有效分类后，在第二个下拉框中选取具体类型，例如 `T cell`。
3. 点击 **“立即检索”**。系统会将检索请求与约束条件一并下发给后端引擎，返回的结果将完全限定于所选属性之中。可结合右侧视图观察跨簇的高维相似关系。
